# 关系图（Diagram View）使用教程

> 适用版本：Phase 5 交付（`@xyflow/react ^12.3` + `@dagrejs/dagre ^1.1`）
> 入口：顶部工具栏 → **视图切换** → **关系图**

关系图把当前项目中的数据表、字段、外键以可拖拽的画布形式呈现，便于审阅模型结构、调整布局，并把布局信息随项目一起持久化。

---

## 1. 打开关系图

1. 顶部工具栏中间的 **Segmented 切换器** 默认选中「设计器」。
2. 点击右侧 **关系图** 即可切换视图，左侧表列表保持可见。
3. 三种空态：
   - 未打开任何项目 → 提示 *尚未加载项目*。
   - 项目内无表 → 提示 *项目中暂无数据表，请先在设计器中新建表*。
   - 项目内有表但无外键 → 正常显示各张表节点，仅没有连线。

> 切回设计器时画布状态保留在内存（节点位置、视口缩放、当前选中表）。

---

## 2. 画布元素速查

### 表节点（TableNode）

```
┌─────────────────────────────┐
│ public.users                │  ← 表头：schema.name，双击跳设计器
├─────────────────────────────┤
│ 🔑 id              BIGSERIAL│  ← 主键 + NOT NULL 红星
│ ★  email *         VARCHAR  │  ← 唯一约束
│ 🔗 role_id         INTEGER  │  ← 外键
│    nickname        VARCHAR  │
└─────────────────────────────┘
```

| 图标 | 含义 |
|------|------|
| 🔑（黄） | 主键（PRIMARY KEY） |
| 🔗（蓝） | 外键（FOREIGN KEY） |
| ★（绿） | 单列 UNIQUE 约束 |
| `*`（红） | NOT NULL |

字段名右侧灰字为 PostgreSQL 类型；数组类型显示为 `INTEGER[]`。
每个字段两侧各有一个连线锚点（Handle），左 = target、右 = source，外键边按字段连接，不是按表中心。

### 外键边（ForeignKeyEdge）

- 灰色直角折线，末端带箭头。
- 中点小徽标 `FK` —— **悬停**显示：
  - 约束名（未设置时显示 `(未命名约束)`）
  - 字段对：`source_field → target_field`
  - `ON DELETE` 动作（`CASCADE / RESTRICT / SET NULL / ...`）
- 边选中态变蓝、加粗。

> 引用已被删除的字段时，对应外键边会被静默隐去，不会出现悬空连线。修复方法：在设计器里把该字段的外键配置改成有效引用或清除。

## 3. 画布交互

| 操作 | 鼠标 / 触控板 |
|------|---------------|
| 平移 | 在空白处按住拖动 |
| 缩放 | 滚轮 / 触控板捏合（缩放范围 0.2x ~ 2x）|
| 拖动表 | 在表头或字段行按住拖动 |
| 选中表 | 单击节点（同步到左侧设计器的"当前表"）|
| 取消选中 | 单击画布空白 |
| 跳到设计器编辑 | **双击表头** |
| 选中外键 | 单击边线 |

左下角的 React Flow **Controls** 提供 `+ / − / fitView` 三个按钮；右下角 **MiniMap** 可点击跳转到对应区域。

---

## 4. 右上角工具栏

### 整理布局

按外键依赖关系用 **dagre LR（左→右）层级算法**重新计算所有节点坐标：

- 被引用的表会被放到更靠左的列。
- 同一层内按节点尺寸自动隔开（`nodesep=60, ranksep=120`）。
- 计算完成后自动 `fitView` 让全部节点可见。

> 整理布局会**覆盖**所有手动摆放的位置；执行后可立即用 `Ctrl+Z` 撤销（前提：撤销栈中存在数据结构变更，仅布局变更目前不进入撤销栈，会被一并丢弃 —— 见 §6）。

### 全屏 / 退出全屏

把画布以 `position: fixed; inset: 0` 浮在最上层，临时遮住其他 UI，便于在大屏上一览大型模型。再次点击恢复。

---

## 5. 布局如何持久化

关系图的所有"位置数据"都保存在 `ProjectFile` 中，跟随项目走，无论保存到本地 JSON 还是数据库 `__dbdesign` 表都会带上：

| 数据 | 字段 | 触发写回时机 |
|------|------|--------------|
| 节点坐标 | `TableDefinition.position` | **拖拽结束** 时（拖拽过程中只更新本地副本，避免高频写入）|
| 画布视口 | `ProjectFile.diagramLayout.{zoom, position}` | **平移 / 缩放结束** 时 |

**触发未保存提示**：节点拖动、整理布局、视口平移缩放都会把 `isDirty` 置为 `true`，工具栏的 `未保存` 橙色 Tag 会亮起。这意味着：

- 直接关闭页面 / 切换项目 → 浏览器不会提醒，但项目状态确实未落盘，建议先「保存」或「存库」。
- 点「导入数据库结构」时会弹「将覆盖当前设计」确认框。
- 点「断开连接」时会弹「未保存改动会丢失」确认框。

**首次打开项目**时：

- 如果 JSON 中带有 `diagramLayout`，画布会立即恢复到上次的视口。
- 否则触发一次 `fitView`，把所有节点居中铺开。

## 6. 与撤销 / 重做的关系

`Ctrl+Z / Ctrl+Y` 只回滚**数据结构性**变更（表、字段、ENUM、索引、外键配置等）。

**布局变更（节点位置、画布视口）不会进入撤销栈**，原因：

- 拖动过程会产生几十次坐标变化，全部入栈会让撤销变得不可用。
- 视口缩放属于"看"的状态，回滚它不会让用户得到符合直觉的结果。

实际表现：

- 拖完一个表 → 改一个字段名 → `Ctrl+Z` → 字段名回滚，节点位置**不会**回到拖动前。
- 想还原布局只能靠：手动重新拖、点「整理布局」，或重新加载 JSON / 从数据库读取。

---

## 7. 与其他视图的联动

| 关系图操作 | 设计器视图反应 |
|------------|----------------|
| 单击节点 | 左侧表列表同步高亮、设计器中央切换到该表（切回设计器时直接看到）|
| 双击节点表头 | **立即切换到设计器**并定位到该表 |
| 单击画布空白 | 取消当前选中表 |

反之，在设计器视图中切换选中表后再切回关系图，对应节点会以蓝色边框 + 浅蓝背景高亮，方便定位。

## 8. 典型工作流

### 8.1 设计阶段

```
新建项目 → 在设计器加表/字段/外键
     ↓
切换到关系图 → 点「整理布局」自动排版
     ↓
微调个别节点位置 → Toolbar 显示「未保存」
     ↓
Ctrl+S 保存到 .dbdesign.json（位置一起落盘）
```

### 8.2 已有数据库逆向

```
连接到目标库 → 点「导入」逆向读取结构
     ↓
进入关系图 → 表节点初始用 4 列网格摆放
     ↓
点「整理布局」按外键依赖重排
     ↓
点「存库」把含布局的设计写入目标库 __dbdesign 表
```

### 8.3 团队评审

```
打开队友发来的 .dbdesign.json
     ↓
关系图 → 全屏模式
     ↓
单击表节点查看主键 / 外键 → 双击节点跳到设计器看字段细节
     ↓
评审完关闭即可（视口若被改动会标 dirty，可选择是否保存）
```

## 9. 常见问题

**Q1：拖动卡顿？**
单一项目里超过 ~100 张表时，React Flow 自身的渲染开销开始可见。临时方案：把不关心的表先在设计器删掉再评审，或用全屏 + 高缩放比让屏幕只渲染部分节点。

**Q2：自动布局后某些边交叉很乱？**
dagre 在循环外键场景下会回退，难免不完美。手动微调几个节点位置即可；调整后**不要**再点「整理布局」，否则会被覆盖。

**Q3：从一个浏览器拖到另一个浏览器后位置没了？**
位置存在 JSON 文件 / 数据库 `__dbdesign` 表里，而不是浏览器本地。确认：
- 文件方案：另存为 `.dbdesign.json` 时 isDirty 是否已落盘。
- 数据库方案：是否点了「存库」而不是只点了「保存」（后者只写本地文件）。

**Q4：自动布局按钮没反应？**
仅当项目中无表时按钮会提示「暂无表可整理」并不执行。其他情况下若没反应请到浏览器 DevTools 控制台查看是否有 React Flow 警告。
